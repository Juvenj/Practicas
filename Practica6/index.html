Resumen

En esta práctica se instaló el entorno de desarrollo local XAMPP, el cual permite simular un servidor web en una computadora personal. A través de esta herramienta se configuró un servidor local utilizando el módulo Apache, asignando un puerto específico para permitir el acceso a través del navegador. Se ejecutaron los servicios con privilegios de administrador para garantizar el correcto funcionamiento del servidor y permitir ajustes en la configuración del sistema. El objetivo fue comprender el proceso de creación y activación de un servidor web básico para desarrollo y pruebas locales.

Introducción

La configuración de un servidor local permite simular un entorno de alojamiento web sin necesidad de recurrir a servidores externos, lo cual es útil en procesos de desarrollo, pruebas o entornos educativos. XAMPP es un paquete gratuito y multiplataforma que integra los servicios esenciales como Apache (servidor web), MySQL (gestor de bases de datos) y PHP, facilitando la creación de aplicaciones web de manera sencilla y rápida. Esta práctica introduce al estudiante en el uso básico de XAMPP y en la comprensión del funcionamiento de un servidor web desde el entorno local.

Desarrollo

La práctica consistió en la instalación, configuración y puesta en marcha de un servidor local utilizando el paquete XAMPP, con el objetivo de alojar un sitio web accesible desde otros dispositivos dentro de la misma red.

  1. Instalación de XAMPP y habilitación de servicios:
    Se descargó el instalador de XAMPP desde su sitio web oficial. Una vez descargado, se ejecutó el instalador como administrador para garantizar los permisos necesarios en el sistema operativo. Tras la instalación, se accedió al Panel de Control de XAMPP, donde se activó el módulo Apache, encargado de gestionar las peticiones HTTP y servir las páginas web.
    Posteriormente, se colocaron los archivos del sitio web dentro de la carpeta htdocs, ubicada en el directorio de instalación de XAMPP. Esta carpeta funciona como la raíz del servidor local y permite que el contenido sea visualizado desde un navegador web.
  2. Configuración del firewall y acceso al puerto 80:
    Dado que Apache utiliza el puerto 80 para atender las solicitudes HTTP, fue necesario revisar la configuración del firewall del sistema operativo para permitir el acceso a dicho puerto. En caso de estar bloqueado, se configuró una excepción en el firewall para permitir el funcionamiento adecuado del servidor y evitar que las conexiones locales fueran rechazadas.
  3. Identificación de la IP del servidor:
    Para que otros dispositivos dentro de la misma red pudieran acceder al sitio web, se identificó la IP local del equipo servidor. Esto se logró utilizando el comando ipconfig en la terminal de Windows, que muestra la dirección IP asignada al dispositivo. Esta IP se utilizó como dirección de acceso desde otros dispositivos (por ejemplo, http://192.168.1.100).
  4. Comprobación de acceso desde otros dispositivos:
    Una vez habilitados todos los servicios y configurado el firewall, se procedió a verificar el acceso al sitio desde otros dispositivos conectados a la misma red. Al introducir la dirección IP del servidor en el navegador, se logró visualizar el contenido web almacenado en htdocs, lo que confirmó el correcto funcionamiento del servidor local y su accesibilidad.

Conclusiones

La práctica permitió comprender de forma clara el proceso de configuración de un servidor local, así como su aplicación para compartir recursos y establecer comunicación entre dispositivos dentro de una red. El uso de XAMPP como herramienta para montar un servidor local facilitó la familiarización con conceptos básicos de administración de redes, servicios web y servidores Apache, elementos fundamentales en el desarrollo web y entornos de prueba. Se destacó la importancia de una correcta configuración de puertos y reglas de firewall, ya que estas determinan la seguridad y accesibilidad del servidor. El desbloqueo del puerto 80 fue clave para permitir el acceso desde otros dispositivos. Finalmente, se comprobó que, al estar correctamente configurado, el servidor local permite a múltiples dispositivos conectados a la misma red acceder al sitio web alojado, lo que demuestra la utilidad de esta práctica en entornos colaborativos, educativos y de desarrollo.
