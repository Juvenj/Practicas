Practica 7: Redes de computadora con Switches y Cables

Abundiz Carbajal Jesus Juventino, Santos Quevedo Sebastian
Ingenieria mecatronica
Fecha: Mayo 15, 2025
WOKWI Electrónica

Resumen

En esta práctica se diseñaron e implementaron diferentes configuraciones de red utilizando tres computadoras, un switch y un router. Se estableció una red alámbrica mediante el uso del switch, una red inalámbrica a través del router y una red combinada que integraba ambos métodos de conexión. Para la conexión alámbrica se emplearon cables Ethernet, y se verificó la conectividad entre los dispositivos realizando pruebas de comunicación mediante el comando ping, utilizando las direcciones IP asignadas automáticamente, obtenidas con el comando ipconfig en la terminal.

Introducción

Las redes de computadoras en entornos locales (LAN) permiten la interconexión de múltiples dispositivos para compartir recursos, servicios e información de forma eficiente. Comprender el funcionamiento y la configuración básica de dispositivos como switches y routers es esencial para el diseño y mantenimiento de redes pequeñas y medianas. Esta práctica permite al estudiante explorar la creación de redes alámbricas, inalámbricas y mixtas, así como aplicar comandos básicos de diagnóstico de red para validar la conectividad.

Desarrollo

En esta práctica se implementaron tres tipos de configuraciones de red utilizando un switch, un router y tres computadoras. Se exploraron redes alámbricas, inalámbricas y combinadas, para entender el comportamiento y funciones de cada dispositivo dentro de una red local.

1. Funcionamiento del switch y del router

  • Switch: Es un dispositivo que opera en la capa 2 del modelo OSI (enlace de datos) y se encarga de conectar múltiples dispositivos dentro de una red local, reenviando los datos al destino correcto según la dirección MAC.
  • Router: Opera en la capa 3 del modelo OSI (red) y conecta diferentes redes entre sí. Además, se encarga de asignar direcciones IP a los dispositivos mediante DHCP y dirigir el tráfico de red entre subredes o hacia Internet.

2. Diseño de red con switch

  1. Se utilizaron cables de red directos para las conexiones entre el switch y las computadoras.
  2. El switch se conectó a tres computadoras, una de las cuales funcionó como servidor local.
  3. Se activó la asignación automática de IP mediante DHCP para cada dispositivo.
  4. Se verificó la conectividad ejecutando el comando ping entre las computadoras y comprobando el acceso al sitio web alojado en el servidor.

3. Diseño de red con router

  1. Se utilizó un router con capacidad para conexiones alámbricas e inalámbricas.
  2. Se conectaron al menos tres computadoras al router, incluyendo una como servidor.
  3. Se configuró el router para asignar direcciones IP de forma automática (DHCP).
  4. Se realizaron pruebas de comunicación mediante ping y se verificó el acceso al servidor desde las demás computadoras.

4. Unión de ambas redes

  1. Se integraron ambas redes (switch y router) utilizando los cables adecuados, según la topología (en algunos casos puede requerirse cable cruzado).
  2. Se mantuvo la asignación automática de IP para todos los dispositivos conectados.
  3. Se comprobó la conectividad global mediante comandos ping y pruebas de intercambio de archivos entre computadoras pertenecientes a ambas redes.

Conclusiones

Comprensión integral de los dispositivos de interconexión: La práctica permitió distinguir claramente el funcionamiento del switch y del router, destacando que, aunque operan en capas distintas del modelo OSI (capa 2 y capa 3, respectivamente), ambos son dispositivos esenciales y complementarios dentro de una red. El switch facilita la segmentación eficiente del tráfico a nivel de direcciones MAC, mientras que el router gestiona el tráfico entre diferentes redes y administra los rangos de direcciones IP. Su correcta configuración evidenció el papel específico que cada uno cumple en la conectividad de red. Importancia de la planificación física y lógica: Seleccionar adecuadamente el tipo de cable (directo o cruzado) y diseñar una topología coherente fueron factores clave para garantizar la conectividad estable entre dispositivos. El uso de cables directos entre hosts y switch, y cruzados para conexiones específicas entre switch y router, demostró la importancia de una planificación precisa tanto del cableado físico como del direccionamiento IP lógico. DHCP como facilitador de escalabilidad y gestión: La asignación automática de IP mediante DHCP simplificó la integración de nuevos dispositivos y redujo errores de configuración. Se evidenció que DHCP es una herramienta fundamental para redes dinámicas o en expansión, permitiendo un crecimiento ordenado y eficiente sin la necesidad de gestionar direcciones manualmente. Verificación y diagnóstico con herramientas básicas: El uso del comando ping y la comprobación del acceso al sitio web del servidor permitieron validar de manera práctica cada etapa de la configuración. Estas pruebas básicas resultaron efectivas para detectar errores de conexión, problemas de cableado o conflictos de IP, demostrando que dominar herramientas sencillas de diagnóstico es fundamental antes de recurrir a soluciones más complejas. Interoperabilidad y transferencia de datos entre subredes: La integración de ambas redes —la alámbrica administrada por el switch y la inalámbrica gestionada por el router— mostró que es posible lograr una infraestructura unificada siempre que se mantenga coherencia en los parámetros de red como el rango de IP, la máscara de subred y la puerta de enlace. Esto refuerza la importancia de diseñar redes modulares con capacidad de escalamiento e integración. Refuerzo de las buenas prácticas de documentación: Documentar topologías, configuraciones, rangos de IP y resultados de pruebas facilitó la detección y resolución de problemas, así como la comprensión del diseño por parte de terceros. Se concluye que la documentación es una herramienta técnica fundamental, no solo para mantenimiento y solución de problemas, sino también para futuras ampliaciones o auditorías de red.
